# MCP Relay Transport — Phase 8: Deletions & E2E Integration

**Goal:** Remove old proxy infrastructure and verify complete relay transport end-to-end with multi-host integration tests.

**Architecture:** Delete the `/api/mcp-proxy` endpoint, `proxyToolCall()`, `generateRemoteMCPCommands()`, `MCPProxyConfig`, and all proxy-related test fixtures. All cross-host MCP calls now use the relay transport exclusively. E2E integration tests verify the complete flow: spoke → hub → target → execute → hub → spoke, for all request kinds and delivery methods.

**Tech Stack:** TypeScript, bun:test, multi-instance test harness

**Scope:** 8 phases from original design (phase 8 of 8)

**Codebase verified:** 2026-03-25

---

### Task 4: E2E integration tests for relay transport

**Verifies:** mcp-relay.AC1.1, mcp-relay.AC1.8, mcp-relay.AC3.1, mcp-relay.AC3.2

**Files:**
- Create: `packages/sync/src/__tests__/relay-e2e.integration.test.ts`

**Testing:**

Create comprehensive end-to-end integration tests using the multi-instance test harness. These tests verify the complete relay transport works for all scenarios.

Follow the patterns from `packages/sync/src/__tests__/multi-instance.integration.test.ts` and the test harness: random ports, unique testRunId, pre-generated keypairs.

**Test scenarios:**

1. **mcp-relay.AC1.1 — Spoke A → Hub → Spoke B tool call:**
   - Set up 3 instances: Hub, Spoke A, Spoke B
   - Spoke B registers a mock MCP tool (via relay processor)
   - Spoke A writes tool_call relay outbox entry targeting Spoke B's tool
   - Spoke A syncs → Hub receives and stores for Spoke B
   - Spoke B syncs → receives tool_call → executes → writes result to outbox
   - Spoke B syncs again → Hub receives result
   - Spoke A syncs → receives result in inbox
   - Verify result matches expected tool output

2. **mcp-relay.AC1.8 — NAT'd host (no sync_url) tool call via polling:**
   - Set up Hub and NAT'd Spoke (no sync_url in hosts table)
   - Write tool_call targeting NAT'd spoke
   - Eager push skipped (no sync_url) — delivery via sync polling only
   - NAT'd spoke syncs → receives and executes → responds
   - Requester syncs → receives result
   - Verify latency is within polling interval bounds (not immediate like eager push)

3. **Addressable host tool call via eager push:**
   - Set up Hub and Spoke with sync_url
   - Write tool_call → Hub eager-pushes to spoke immediately
   - Spoke receives via /api/relay-deliver (INSERT OR IGNORE)
   - Spoke processes and writes response
   - Verify delivery was via eager push (check relay_cycles metrics)

4. **mcp-relay.AC3.1 + AC3.2 — Proxy endpoints removed:**
   - POST to `/api/mcp-proxy` → 404
   - POST to `/api/file-fetch` → 404

5. **Hub migration with active relay:**
   - Hub A, Hub C, Spoke B
   - Spoke B has active relay outbox entries
   - Initiate hub switch with drain
   - Verify drain completes, held entries deliver to new hub

6. **Idempotent retry:**
   - Send same relay request twice with same idempotency_key
   - Verify tool executes only once
   - Both requests receive same result

7. **Resource read E2E round-trip:**
   - Spoke A requests resource_read targeting Spoke B
   - Spoke B has MCP client with the resource
   - Full relay round-trip → Spoke A receives resource content
   - Verify content matches expected resource

8. **Prompt invoke E2E round-trip:**
   - Spoke A requests prompt_invoke targeting Spoke B
   - Spoke B has MCP client with the prompt
   - Full relay round-trip → Spoke A receives prompt result
   - Verify result messages match expected prompt output

9. **Cache-warm with multi-file split:**
   - Spoke A requests cache_warm for multiple files targeting Spoke B
   - Spoke B reads files, total exceeds 2MB → splits into per-file result messages
   - Full relay round-trip → Spoke A receives and reassembles all file contents
   - Verify reassembled content matches original files

**Verification:**

Run: `bun test packages/sync/src/__tests__/relay-e2e.integration.test.ts`
Expected: All tests pass.

Run: `bun test --recursive`
Expected: All tests across all packages pass.
